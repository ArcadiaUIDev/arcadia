# Blog Writing Rules — Anti-AI Slop

## Banned Phrases (never use these)
- "In today's fast-paced world"
- "Let's dive in" / "dive deep"
- "Without further ado"
- "In this comprehensive guide"
- "It's worth noting that"
- "At the end of the day"
- "Game-changer" / "Revolutionary"
- "Leveraging" / "Harnessing the power of"
- "Seamlessly" / "Effortlessly"
- "Robust" (unless describing actual error handling)
- "Cutting-edge" / "State-of-the-art"
- "Take your X to the next level"
- "Whether you're a beginner or expert"
- "In conclusion" (just end the article)

## Structure Rules
- Lead with code or a concrete result, not a preamble
- First 100 words must contain a code block or specific technical claim
- No "What is Blazor?" sections — readers know what Blazor is
- Every code block must be verified to compile
- Short paragraphs (3-4 sentences max)
- Use headers as scannable answers to questions
- Be opinionated — "X is better than Y because Z"
- Include specific numbers (render times, bundle sizes, test counts)
- End with a CTA, not a summary

## Code Rules
- Every code sample must exist in a verified test or sample project
- Include the @using statements — don't make readers guess
- Show the complete working example, not fragments
- If referencing data, show the data model too

## SEO Rules
- Primary keyword in H1, first paragraph, and 2-3 H2s
- Secondary keywords in H2s and body naturally
- Meta description under 160 chars, includes primary keyword
- URL slug contains primary keyword
- Internal links to docs pages
- External links to 2-3 authoritative sources
